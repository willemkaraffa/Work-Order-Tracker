## What changed and why

### Chrome extension — `tracker chrome extension/wo-extension/`
- `manifest.json` — bumped 2.0.0 → 2.1.0.
- `content.js`
  - `extractWONumber()` rewritten with title/heading/label/URL/body strategies (was missing portal WO# in many cases).
  - New `extractPropertyId()` — labeled fields, URL params (`?propertyId=`, `/property/`), body text fallback. Targets AMH NC####### values.
  - `scrapeAMH()` now sets `data.propertyId` and `data.portalLink` (= current URL).
  - `scrapeMSR()` now sets `data.portalLink`.
  - Capture toast shows Prop ID alongside WO#, Address, Priority, Phone.

### Tracker — `WO_Tracker-Source/index.html`
- `confirmImport()` uses `item.woId` as the tracker `id` field when present (with collision fallback to sequential WO-###). Was generating new WO-### regardless of portal value.
- Import modal now displays WO #, Property ID, and portal link for each captured row.
- `DSETTINGS` adds `workbookPath:''`.
- `renderSettingsModal()` adds workbook path input + Browse + Clear buttons.
- New `chooseWorkbook()` renderer function bridges to `window.workbook.choose()`.
- `syncWorkbook()` now passes `S.settings.workbookPath` to the IPC bridge.

### Tracker — `WO_Tracker-Source/main.js`
- New `resolveWorkbookPath(overridePath)` — explicit setting wins; falls back to OneDrive\Desktop\WORK ORDERS, then exe dir (legacy), then dev install dir.
- `sync-workbook` IPC handler accepts override path; returns clear error if workbook missing with instruction to set in Settings.
- New `choose-workbook` IPC handler — opens native file picker filtered to .xlsx/.xlsm; defaults to last-used dir or Desktop\WORK ORDERS.

### Tracker — `WO_Tracker-Source/preload.js`
- `window.workbook.sync(overridePath)` — now accepts arg.
- `window.workbook.choose(currentPath)` — new bridge for the file picker.

### Tracker — `WO_Tracker-Source/sync_to_lookup.py`
- Rewritten to write to **Invoice Import** sheet (was Lookup).
- Layout: `Invoice # | Customer Name | Service Address | Date | Item/Service | Unit Price | Tax | Memo`
- Customer Name maps `AMH → American Homes 4 Rent`, `MSR → Main Street Renewal`, else uses `pm` verbatim.
- Date column intentionally blank.
- Item/Service blank (manual selection from dropdown sourced from Service Items library).
- Unit Price uses `bidAmount` when present, else blank.
- Tax column gets a VLOOKUP formula that pulls from Service Items library by Item/Service.
- Memo: `WO #######` for non-AMH; `WO ####### | PropID: NC#######` for AMH. Skips redundant "WO " prefix when id already starts with "WO".
- Auto-detect path order: `%USERPROFILE%\OneDrive\Desktop\WORK ORDERS\` → `%LOCALAPPDATA%\Programs\Work Order Tracker\` → script dir.

### Tracker — `WO_Tracker-Source/package.json`
- `build.publish.releaseType: "release"` added — prevents drafts. (Saved as memory; applies to all Electron projects going forward.)
