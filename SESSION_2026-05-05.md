# Session Log - 2026-05-05

Five focused changes to the tracker, extension, and workbook sync.

## Versions shipped

| Version | Component | Summary |

| Extension 2.2.0 | Chrome extension | AMH WO# regex constrained to leading 7 digits; City field added |

## Change 1 - Status display + workbook sync rewrite

### Tracker (`index.html`)
- `DSTATUSES` no longer contains `Submitted for Invoice`. `DSC` keeps the color for legacy data plus `Paid in Full`.
- `displayStatus(o)` returns `Submitted for Invoice` when `o.tab==='invoiced'` and `Paid in Full` when `o.tab==='paid'`. Stored `o.status` is untouched.
- Status cell on Invoiced/Paid tabs renders as a static badge - cycle and edit dropdown disabled there.
- Peek pane uses `displayStatus`.
- Load-time migration prunes legacy `Submitted for Invoice` and `Paid in Full` entries from `S.statuses`.

### Bridge (`main.js`)
- `/config` default statuses updated to match the new pickable list.

### Sync (`sync_to_lookup.py`)
- Filters to `tab==='invoiced'` only (Active, Paid, deleted excluded).
- New 7-column layout: `Invoice # | Customer Name | Service Address | Date | Item/Service | Unit Price | Memo`. Tax column dropped.
- Per WO: anchor row carries Invoice/Customer/Address/Memo + first Item/Service slot, followed by one blank separator row.
- Unit Price = `=IFERROR(IF(En="","",VLOOKUP(En,'Service Items'!A:C,3,FALSE)),"")` filled across rows 2-2000 so user-inserted rows auto-fill.
- AMH memo: `WO ####### | NCXXXX` (no `PropID:` prefix).

### One-shot workbook upgrade (`rebuild_invoice_import_sheet.py`, NEW)
- Recreates Invoice Import + README tabs with new layout. Service Items library preserved. Run once per machine.

### Tested
- 6 synthetic WOs (Active/Invoiced/Paid/deleted mix) -> 3 Invoiced rows written, others excluded.
- VLOOKUP returns 75/55/400 for representative items. Zero formula errors via LibreOffice recalc.
- Data validation present: Item/Service E2:E2000 from `ServiceItemNames`; Customer Name B2:B2000.

## Change 2 - AMH WO# extra-digits bug

### Extension (`content.js`)
- `extractWONumber()` now branches on `isAMHPage()`. AMH path captures any `\d{7,}` run but slices to the leading 7 digits, fixing the `9698891706 -> 9698891` case.
- MSR path keeps original `\d{5,}` heuristics.
- `manifest.json` 2.1.0 -> 2.1.1.

### Tested
- 10/10 unit tests pass (concatenated runs, clean WO#, headings, body keyword, ambient long digits, UUID URL, MSR fallback).

## Change 3 - City field

### Extension (`content.js`)
- New `extractCityFromAddress(addr)` walks comma-segmented address from the right, skipping zip-only / state-only / "STATE ZIP" segments, returns the city.
- Both `scrapeAMH` and `scrapeMSR` populate `data.city`.
- Capture toast surfaces `City: Apex` line.

### Tracker (`index.html`)
- Schema migration adds `city` (default `''`).
- `confirmImport` carries `city` through.
- Import preview shows city tag alongside Prop ID.
- Peek pane gains a City row.
- Search keys include `city`.
- Add/Edit modal gains a City input.
- CSV export `cols` array includes `city`.

### Tested
- 10/10 city-extractor unit tests pass (NC long-form, abbreviated state, zip+4, missing city, empty, malformed).

## Change 4 - UI cleanup: manage widgets into Settings

### Tracker (`index.html`)

- Settings modal gains a "Lists" section with the same five buttons.
- `openManage` sets `S._manageReturnsToSettings=true` when launched from Settings.

### Tested

## Change 5 - Emergency/Warranty flags, deprecate priority tags

### Tracker (`index.html`)
- New schema fields: `emergency:false`, `warranty:false` (booleans, migrated on load).
- `DSETTINGS.warrantyDisplay='flag'` (alternative `'highlight'`); Settings modal exposes a two-button toggle.
- Row rendering:
  - Priority-color row tint removed.
  - Red flag (Unicode 0x2691) rendered next to WO# when `emergency=true`.
  - Blue flag rendered next to WO# when `warranty=true` AND `warrantyDisplay==='flag'`.
  - When `warrantyDisplay==='highlight'`, warranty=true applies a blue row tint instead.
- Peek pane gains the same two flag indicators (in place of the old priority badge).
- Add/Edit modal: two checkbox rows (Emergency, Warranty) with colored flag glyphs as labels.
- New `toggleFlag(id, field)` toggles either flag from the row; logs to history.
- Order modal `saveOrder` reads the new checkboxes and the City input.
- CSS: `.wo-flag`, `.wo-flag-emerg` (#ef4444), `.wo-flag-warranty` (#1a73e8).

### Tested
- 10/10 flag rendering assertions pass across both display modes (flag-only, highlight-only, combinations).
